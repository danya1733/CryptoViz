# CryptoViz - Визуализатор Криптовалютных Транзакций

CryptoViz - это веб-приложение для визуализации и анализа криптовалютных транзакций. Приложение позволяет пользователям отслеживать и визуализировать движение различных криптовалют (USDT, Bitcoin, Ethereum) в виде интерактивных графов.

## Функциональность

- **Визуализация транзакций**: Построение графов транзакций между криптовалютными кошельками
- **Поддержка нескольких криптовалют**: Bitcoin, Ethereum, USDT (TRC20, ERC20)
- **Фильтрация по параметрам**: 
  - По дате транзакций
  - По сумме транзакций
  - По глубине связей между кошельками
- **Анализ графа**:
  - Расчет количества адресов, переводов, компонент связности и суммарных объемов
  - Расчет свойств выбранной вершины: входящие и исходящие переводы, уникальные контрагенты, основной получатель
  - Выделение возможных депозитных адресов сервиса, возможных сервисных адресов биржи или обменника, обычных адресов и дополнительных связующих признаков
  - Дополнительная проверка масштаба активности заметных TRC20-адресов по количественным счетчикам Tronscan
- **Персонализация**: 
  - Добавление меток к кошелькам
  - Настройка иконок для кошельков (обмен, личный, бизнес и т.д.)
- **Управление кошельками**: Добавление и удаление отслеживаемых кошельков
- **Авторизация**: Система регистрации и входа для сохранения персональных настроек
- **Экспорт/импорт данных**: Возможность сохранения и загрузки данных о транзакциях

## Технический стек

### Frontend
- **React**: Библиотека для создания пользовательского интерфейса
- **Vite**: Инструмент сборки и разработки
- **vis-network**: Библиотека для визуализации графов
- **React Router**: Маршрутизация в приложении
- **React DatePicker**: Компонент для выбора даты

### Backend
- **Node.js**: Среда выполнения JavaScript
- **Express**: Веб-фреймворк для Node.js
- **Sequelize**: ORM для работы с базой данных
- **PostgreSQL**: Реляционная СУБД для хранения данных
- **JWT**: Аутентификация и авторизация пользователей
- **bcrypt**: Хеширование паролей
- **nodemailer**: Отправка электронных писем (для восстановления пароля)

### Blockchain Explorer APIs
- **Blockchair API**: Для получения данных о Bitcoin транзакциях
  - Endpoint: `api.blockchair.com`
  - Используется для получения списка транзакций и детальной информации о каждой транзакции
- **Etherscan API**: Для получения данных о Ethereum и ERC-20 токенах
  - Endpoint: `api.etherscan.io`
  - Требует API ключ для аутентификации
  - Поддерживает получение ETH транзакций и ERC-20 токен-трансферов (USDT, USDC)
- **Tronscan API**: Для получения данных о TRON TRC-20 токенах
  - Endpoint: `apilist.tronscan.org`
  - Используется для получения USDT (TRC-20) транзакций
  - Запросы выполняются через клиентскую очередь с паузой и обработкой ограничения `429`

## Интеграция с Blockchain Explorer APIs

CryptoViz интегрируется с несколькими ведущими обозревателями блокчейна для получения данных о транзакциях:

### Tronscan API (TRON Network)
- **URL**: `https://apilist.tronscan.org/api/token_trc20/transfers`
- **Назначение**: Получение TRC-20 токен-трансферов (USDT на сети TRОН)
- **Параметры**: 
  - `relatedAddress` - адрес кошелька
  - `limit` - количество записей
  - `sort` - сортировка по времени
- **Особенности**: Не требует API ключа, бесплатный доступ
  - В приложении используется общий клиент запросов с очередью, чтобы не отправлять несколько запросов к Tronscan одновременно

### Blockchair API (Bitcoin Network)
- **URL**: `https://api.blockchair.com/bitcoin/dashboards/`
- **Назначение**: Получение данных о Bitcoin транзакциях
- **Endpoints**:
  - `/address/{address}` - список транзакций для адреса
  - `/transaction/{hash}` - детальная информация о транзакции
- **Особенности**: 
  - Бесплатный доступ с ограничениями
  - Предоставляет полную информацию о входах и выходах транзакций
  - Поддержка виртуальных узлов для сложных транзакций

### Etherscan API (Ethereum Network)
- **URL**: `https://api.etherscan.io/api`
- **Назначение**: Получение данных о Ethereum и ERC-20 токенах
- **Endpoints**:
  - `module=account&action=txlist` - ETH транзакции
  - `module=account&action=tokentx` - ERC-20 токен-трансферы
- **Особенности**:
  - Требует регистрации и получения API ключа
  - Поддержка множественных ERC-20 токенов (USDT, USDC)
  - Лимиты запросов в зависимости от тарифного плана

## Структура базы данных

Приложение использует PostgreSQL для хранения пользовательских данных. Все таблицы автоматически включают системные поля `id` (первичный ключ), `createdAt` и `updatedAt` (временные метки).

### Основные таблицы и их структура:

#### 1. **Users** - Основная таблица пользователей
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор пользователя
- `username` (STRING, UNIQUE, NOT NULL) - Имя пользователя
- `email` (STRING, UNIQUE, NOT NULL) - Электронная почта
- `password` (STRING, NOT NULL) - Хешированный пароль
- `createdAt` (DATETIME) - Дата создания записи
- `updatedAt` (DATETIME) - Дата последнего обновления

**Связи:**
- Один ко многим с `UserWallets` (пользователь может иметь множество кошельков)
- Один ко многим с `UserWalletImages` (пользователь может настроить иконки для множества кошельков)
- Один ко многим с `UserWalletLabels` (пользователь может добавить метки к множеству кошельков)
- Один ко многим с `PasswordResetTokens` (пользователь может иметь несколько токенов сброса)
- Один к одному с `UserApiKeys` (у пользователя может быть один API ключ)

#### 2. **UserWallets** - Кошельки пользователей
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор записи
- `userId` (INTEGER, FOREIGN KEY) - Ссылка на пользователя
- `walletAddress` (STRING, NOT NULL) - Адрес криптовалютного кошелька
- `cryptoType` (STRING(10), NOT NULL) - Тип криптовалюты (BTC, ETH, USDT-TRC20, USDT-ERC20)
- `createdAt` (DATETIME) - Дата добавления кошелька
- `updatedAt` (DATETIME) - Дата последнего обновления

**Индексы:**
- Уникальный составной индекс по (`userId`, `walletAddress`, `cryptoType`)

**Связи:**
- Многие к одному с `Users` (множество кошельков принадлежат одному пользователю)

#### 3. **UserWalletImages** - Пользовательские иконки кошельков
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор записи
- `userId` (INTEGER, FOREIGN KEY) - Ссылка на пользователя
- `walletAddress` (STRING, NOT NULL) - Адрес кошелька
- `imageType` (STRING(20)) - Тип иконки (exchange, general, home, salary, transaction)
- `cryptoType` (STRING(10)) - Тип криптовалюты
- `createdAt` (DATETIME) - Дата создания записи
- `updatedAt` (DATETIME) - Дата последнего обновления

**Индексы:**
- Уникальный составной индекс по (`userId`, `walletAddress`, `cryptoType`)

**Связи:**
- Многие к одному с `Users` (множество настроек иконок принадлежат одному пользователю)

#### 4. **UserWalletLabels** - Пользовательские метки кошельков
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор записи
- `userId` (INTEGER, FOREIGN KEY) - Ссылка на пользователя
- `walletAddress` (STRING, NOT NULL) - Адрес кошелька
- `label` (STRING) - Пользовательская метка для кошелька
- `cryptoType` (STRING(10)) - Тип криптовалюты
- `createdAt` (DATETIME) - Дата создания записи
- `updatedAt` (DATETIME) - Дата последнего обновления

**Индексы:**
- Уникальный составной индекс по (`userId`, `walletAddress`, `cryptoType`)

**Связи:**
- Многие к одному с `Users` (множество меток принадлежат одному пользователю)

#### 5. **PasswordResetTokens** - Токены для восстановления пароля
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор записи
- `userId` (INTEGER, FOREIGN KEY) - Ссылка на пользователя
- `token` (STRING, UNIQUE) - Уникальный токен для сброса пароля
- `expiresAt` (DATETIME) - Дата истечения срока действия токена
- `createdAt` (DATETIME) - Дата создания токена
- `updatedAt` (DATETIME) - Дата последнего обновления

**Связи:**
- Многие к одному с `Users` (пользователь может запросить несколько токенов сброса)

#### 6. **UserApiKeys** - API ключи пользователей
**Столбцы:**
- `id` (INTEGER, PRIMARY KEY) - Уникальный идентификатор записи
- `userId` (INTEGER, FOREIGN KEY) - Ссылка на пользователя
- `apiKey` (TEXT) - API ключ для внешних сервисов (например, Etherscan)
- `createdAt` (DATETIME) - Дата создания записи
- `updatedAt` (DATETIME) - Дата последнего обновления

**Индексы:**
- Уникальный индекс по `userId` (один пользователь - один API ключ)

**Связи:**
- Один к одному с `Users` (у пользователя может быть только один API ключ)

### Схема связей между таблицами:

```
Users (1) ────────────── (∞) UserWallets
  │                           └── userId → Users.id
  │
  ├─────────────────── (∞) UserWalletImages  
  │                           └── userId → Users.id
  │
  ├─────────────────── (∞) UserWalletLabels
  │                           └── userId → Users.id
  │
  ├─────────────────── (∞) PasswordResetTokens
  │                           └── userId → Users.id
  │
  └─────────────────── (1) UserApiKeys
                              └── userId → Users.id
```

### Ограничения целостности:
- При удалении пользователя (`CASCADE`) автоматически удаляются все связанные записи
- Уникальность комбинации пользователь-кошелек-тип_криптовалюты в таблицах кошельков, иконок и меток
- Уникальность username и email в таблице Users
- Уникальность токенов сброса пароля
